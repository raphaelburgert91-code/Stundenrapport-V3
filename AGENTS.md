# AGENTS.md

## Arbeitsweise

- Zuerst einen kurzen Plan erstellen.
- Änderungen in kleinen Schritten durchführen.
- Bestehenden funktionierenden Code nicht unnötig verändern.
- Bei Unsicherheiten nachfragen.

## Code

- Clean Code bevorzugen.
- Keine unnötigen Bibliotheken verwenden.
- Quellcode verständlich und wartbar halten.
- Kommentare auf Deutsch, Variablen und Funktionen möglichst klar benennen.

## Benutzeroberfläche

Priorität:

1. Robuste Funktion
2. Einfache Bedienung
3. Tablet-Optimierung
4. Wartbarkeit
5. Modernes Design

## Git

- Nach abgeschlossenen Funktionen einen Commit vorschlagen.
- Commit-Nachrichten auf Englisch.

## Dokumentation

- README aktuell halten.
- CHANGELOG bei größeren Änderungen ergänzen.
- Projektstruktur in diesem Repository bewusst pflegen.
